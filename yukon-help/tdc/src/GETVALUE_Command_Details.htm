GETVALUE Command Details

This command is used to collect metering type data from the various supported field devices. Most field device types support multiple metering quantities (kWh, current kW, peak kW, kVarh, current kVar, etc.). This command is also used to collect load control information from two-way LCRs such as the LCR 3102. To allow the collection of the this data, the GETVALUE command supports the following arguments.

 

getvalue kwh [frozen]
getvalue daily read [channel (1|2|3)] mm/dd/yyyy [mm/dd/yyyy]

getvalue [[ied] (kwh|kvarh|kvah) [a|b|c|d|t] [update]] [frozen] [rate (a|b|c|d|t)] [update]]

getvalue [[ied] (kwh|kvarh|kvah) [rate (a|b|c|d|t)] [channel channel] [update]] [frozen]

getvalue [[ied] (kw|kva|kvar|demand|peak|voltage) [offset channel] [update]] [frozen]

getvalue [ied dnp (collection [1-2]|status|analog [1-5]|accumulator [1-4])]

getvalue usage
getvalue [lp channel (1|2|3|4) date time] [status] [cancel] [background]

getvalue [lp peak (day|hour|interval) channel (1|2|3|4) mm/dd/yyyy (days)]

getvalue [minmax|peak [update]] [frozen]

getvalue [powerfail] [feeezecount]

getvalue [outage (1|2|3|4|5|6)]

getvalue interval last

getvalue runtime load (1|2|3|4) previous (12|24|36)

getvalue shedtime relay (1|2|3|4) previous (12|24|36)

getvalue propcount

 

The arguments are as follows:

 

Argument

Description

daily read

Collects daily self-read data from a field device for a date or a date range that is within the past 3 months. If a date range is specified, only the consumption values for the range are returned. Defaults to channel 1 if a channel is not specified.

  Channel 1: kWh at day end, peak kW, daily min./max. voltage

  Channel 2: consumption (if defined), peak daily demand (if defined), power outages to date

  Channel 3: consumption (if defined), peak daily demand (if defined)

ied

Collects IED Port data from field device.

kwh

Collects the energy reading from a field device.

kvarh

Collects the reactive reading from a field device.

kvah

Collects the voltage-amp reading from a field device.

[a|b|c|d|t]
rate (a|b|c|d|t)

Collects the TOU accumulator and current or frozen peak for the specified rate.

channel #

Specifies the pulse input channel from which to collect data. Defaults to channel 1 if no channel is specified.

update

Saves the data into the current/last reading table.

usage

Collects kWh data for all channels from a field device.

frozen

Collects frozen data instead of current data.

kw

Collects the current kW from a field device.

kva

Collects the current kVa from a field  device.

kvar

Collects the current kVar from a field  device.

demand

Collects the demand reading from a field device.

peak

Collects the peak reading from a field device.

voltage

Collects the minimum and maximum voltage readings from a field device.

offset channel

Specifies the channel from which to collect data.

ied dnp
(collection [1-2]|
status|
analog [1-5]|
accumulator [1-4])

Returns DNP data when using a DNP enabled MCT-470. The collection option reads out of the real time tables and is a combination of status, analog, and accumulator data.

lp channel (1|4) date time

Collects specific blocks or data ranges of Load Profile data from a field device.

lp peak (day|hour|interval) channel (1|2|3|4) mm/dd/yyyy (days)

Collects a peak usage report of a field device’s load profile data. The command can return the peak day, hour, or interval. The day|hour|interval parameter specifies how far back the report should go.

status

Returns the status of a load profile read.

cancel

Cancels a load profile read.

background

Results of the command are not returned to the Yukon Commander application.

minmax

Collects the minimum and maximum values from a field device.

powerfail

Collects the powerfail count from a field device.

freezecount

Collects the freeze counter from a field device.

outage (1|2|3|4|5|6)

Collects the specified outage logs. Specifying 1 or 2 collects outage logs 1 and 2. Specifying 3 or 4 collects outage logs 3 and 4. Specifying 5 or 6 collects outage logs 5 and 6.

interval last

Returns the amount of kW used by the LCR in the last interval. Data is returned for all the relays with this command.

runtime load (1|2|3|4) previous (12|24|36)

Returns the amount of run time for a load (1, 2, 3, or 4) for each hour over the time period (previous 12, 24, or 36 hours) that was requested. This data is also used by the TrueCycle gear. The value returned is in minutes.

shedtime relay (1|2|3|4) previous (12|24|36)

Returns the amount of shed time for a relay (1, 2, 3, or 4) for each hour over the time period (previous 12, 24, or 36 hours) that was requested. The value returned is in minutes.

propcount

Increments and returns the value in the propagation counter. The propagation counter keeps track of the number of times the LCR received a command.
