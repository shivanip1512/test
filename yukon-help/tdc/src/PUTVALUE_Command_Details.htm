PUTVALUE Command Details

This command is used to send metering type data to the various supported field devices. To support the different available functions, the PUTVALUE command supports the following arguments.

 

putvalue analog offset value

putvalue ied reset [alpha|s4|sentinel]

putvalue kyz offset [reset|reading value]

putvalue ovuv analog (1 1|0 1)

putvalue powerfail reset

putvalue tou reset [zero]
 

The meanings of the arguments are as follows:

 

Argument

Description

analog offset value

Writes the value to the specified analog output point on a DNP device.

ied reset [alpha|s4|sentinel]

Resets the demand reading for an IED attached to a field device. When the alpha, s4, or sentinel option is used, the command performs a freeze and reset on the IED’s current peak demand.

kyz offset  [reset|reading value]

Specifies the KYZ offset to read or reset.

ovuv analog (1 1|0 1)

Enables (1 1) or disables (0 1) Over Voltage (OV) Under Voltage (UV) for CBCs.

powerfail reset

Resets the powerfail count.

tou reset [zero]

Copies the current MCT readings into the default TOU rate and clears the other rates. If the zero option is used, the command clears all TOU rates.

 

Examples

putvalue analog 5 32

putvalue ied reset alpha

putvalue ied reset s4

putvalue ied reset sentinel

putvalue kyz 1 reset

putvalue kyz 1 reading 297

putvalue ovuv analog 1 1

putvalue ovuv analog 1 0

putvalue powerfail reset

putvalue tou reset

putvalue tou reset zero
