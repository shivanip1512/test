Alarms and Events

To view alarms and events:

 

  1. From the Display menu, select Alarms and Events. The number of unacknowledged alarms appears in the upper right-hand corner of the window.

  2. Click on Unacked Alarms in the upper right hand corner of the TDC window to display a drop-down list of the last three unacknowledged alarms. Selecting an alarm from the drop-down list displays the alarm’s Category page.

  3. Select a view to display from the View drop-down list. The views include:

    • All Alarms - displays all active and unacknowledged alarms. See All Alarms and Alarm Categories for more information.

    • Event Viewer - displays all Yukon events for the day selected from View Date drop-down.

    • Category 1 to Category x - displays active and unacknowledged alarms for the selected category.

    • SOE Log - displays data for points on DNP devices that have their timing group set to SOE (Sequence of Events). That data that is recorded are high speed events that occur within milliseconds of each other.

    • TAG Log - displays information about tags that have been assigned to devices.

  4. Click on any of the toolbar buttons for the following functionality:

    • Silence - turns off the audible alarm for all current alarms. The alarms continue to flash until they are acknowledged.

    • Mute/Unmute - mute turns off the audible alarm for all current and future alarms. Unmute turns the audible alarm back on for all current and future alarms. The alarms continue to flash until they are acknowledged.

    • Refresh - refreshes the data on the current display.

    • Clear Display - clears the display of past events.

    • Ack Viewable - acknowledges all viewable alarms. Alarms stop flashing and the audible alarms stop. However, alarms still display in the table until the points that caused the alarms change to a non-alarming state.

    • Filter - the filter drop-down list lets you display one of the following:

      • No Filter - the points displayed are all the active and unacknowledged alarms.

      • Active Alarms - the points displayed are currently in an alarm state.

      • Alarms History - the points displayed are the alarms that have occurred on the day selected from the View Date drop-down.

      • Unacked. Alarms - the points displayed are in an alarm state, which has not been acknowledged, or the points have moved out of an alarm state and are waiting acknowledgement.

  5. Right-click on an alarm to display a menu with the following functionality:

    Ack Alarm     - displays a list of active alarms for the point, from which you can acknowledge them. If more than one alarm state is active for the point, the option (ACK ALL) appears in the list. When an alarm is acknowledged, the alarm stops flashing and the audible alarms stop. However, the alarm still displays in the table until the point that caused the alarm changes to a non-alarming state.
