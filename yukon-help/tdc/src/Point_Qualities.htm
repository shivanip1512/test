Point Qualities

Quality

Meaning

Definition

Uninit

Unintialized

Assigned to a point for which no additional data is known. This quality is assigned to points when they are first added to a display. Points keep this quality until their values are scanned or acquired for the first time. A point with an uninitialized quality has a change date of 1/1/1990.

Non

Non-Updated

The device failed a scan. Each point on the device is assigned this quality. The time of the failed scan appears in the Time column. This is an alarmable condition.

Man

Manual

A user manually changed the value in the system via TDC (or another client).

Norm

Normal

A normal value.

Abnormal

Abnormal

Used by Foreign Data Routing (FDR) to indicate an abnormal quality condition from the remote system.

Unknown

Unknown

Used by FDR and IDLC type devices to indicate a unknown quality.

Part-Int

Partial Interval

A point on the device only recorded part of the data for this interval. This typically occurs after a power outage, when the device turns back on in the middle of an interval.

Dev-Fill

Device Filler

A point on the device was unable to record data for the entire interval. This typically occurs because of a power outage. The device assigns all missed intervals with the value “0” and this quality. For power outages longer than a single interval, the Dev-Fill quality appears immediately before the Part-Int quality.

Quest

Questionable

Value received by FDR is not Normal or Manual. This is used to catch all of the other possible qualities a foreign system may send Yukon.

Overflow

Overflow

A point on a device collected too many pulses during the interval. This typically means the device's internal multiplier is set incorrectly.

Pow-Fail

Power Fail

A point on a device is assigned this quality when the device first loses power.

Unr

Unreasonable

Inbound point data exceeds the editor specified reasonability limits. The point is assigned this quality and a current timestamp. The point also retains the old data value.
