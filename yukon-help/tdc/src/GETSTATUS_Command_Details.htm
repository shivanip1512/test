GETSTATUS Command Details

This command is used to collect status type data from the various supported field devices. To allow the collection of the different statuses, the GETSTATUS command supports the following arguments.

 

getstatus [lp] [disconnect] [internal|external] [ied [link]] [errors] [pfcount] [eventlog] [update] [freeze]

 

The arguments are as follows:

 

Argument

Description

lp

Collects state information regarding current Load Profile data collection.

disconnect

Collects the state of the disconnect for a meter equipped with a whole house disconnect.

internal

Collects status internal to the device. For example, the firmware status.

external

Collects status external to the device. For example, a status input.

ied [link]

Collects a status that indicates whether or not an IED is linked to the device.

errors

Collects any internal error or state flags (power fail or firmware flags).

pfcount

Collects the power fail count from a metering type device.

eventlog

Collects the eventlog for ION devices only.

update

Saves the data into the current/last reading table.

freeze

Reads the last freeze time stamp and the current freeze counter data.

Examples

getstatus errors

getstatus eventlog

getstatus freeze

getstatus lp

getstatus lp update

getstatus disconnect

getstatus internal

getstatus external

getstatus freeze

getstatus ied

getstatus ied link

getstatus pfcount
