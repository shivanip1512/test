PUTCONFIG Command Details

This command is used to send configuration type data to the various supported field devices. To support the different available functions, the PUTCONFIG command supports the following arguments.

ExpressCom

putconfig xcom [sync] [setstate [relaymask #] [run] [hold] [fan (on|off|auto)] [system (off|heat|cool|emheat)] [temp value] [timeout time]] [timesync date hh:mm] [template name] [raw #] [data #] [maintenance #] [assign|address [spid #] [geo #] [sub #] [feeder #] [zip #] [uda #] [program #] [splinter #]] [relay|load #] [service (in|out|enable|disable) [t] (noclp|noled|bitP|bitL|cancel| offhours time)] [schedule day HH:MM, Heat Temp, Cool Temp, ...] [data ‘text message’ port # deletable msgpriority # timeout # hour clear] [extended tier # rate # cmd ‘cmd’ display # timeout # delay #] [utility info chan # [cost [cents]] [usage] “optional text”] [utility usage channel:past usage:value,channel:present usage:value,channel:past cost:value,channel:present cost:value]

[serial # targetamps #]

 

The arguments are as follows:

 

Argument

Description

xcom

Target type of putconfig message.

sync

Sync message that keeps devices locked to a paging frequency and cap code.

setstate

Sets the thermostat state.

relaymask

Specifies the relays of the device.

run

Runs a program

hold

Holds the current temperature.

fan (on|off|auto)

Sets the fan state to on, off, or auto.

system (off|heat|cool|emheat)

Sets the system state to off, heat, cool, or emheat (heat pump).

 

temp value

Sets the temperature.

timeout time

Sets the length of time the program runs until it times out.

timesync date hh:mm

Sends a time sync with the current time of day. Including a date sends a the current time and date. Specifying the hours and minutes sends that particular time.

template name

Configures a device to use the addressing of the specified Load Group. Double quotes are required if the name of the load group contains spaces.

raw #

Sends an uninterpreted configuration command to the device. Contact Cooper Power Systems Customer Service for more information.

data #

Sends an uninterpreted data message to the device. Contact Cooper Power Systems Customer Service for more information.

maintenance #

Sends an uninterpreted maintenance function to the device. Contact Cooper Power Systems Customer Service for more information.

assign [s #] [g #] [b #] [f #] [z #] [u #] { [p #] [r #] [load #] }

Assigns a new ExpressCom® address to the device. The address must be entered in the order listed.

s = Spid

g = Geographic

b = suBstation

f = Feeder

z = Zip code

u = User-defined

p = Program

r = splinteR

service
(in|out|enable|disable) [temp] (noclp|noled|offhours time)

Sends a command to take the device out of service (disable) or to place the device in service (enable). The temp option makes the service command a temporary command. The noclp option keeps the device from running cold load pickup during startup. The noled option turns of the LEDs when the device is out of service. The offhours time option takes the device out of service for a specified number of hours.

schedule day HH:MM, Heat Temp, Cool Temp, ...

Assigns period schedules to a thermostat.

data ‘text message’ port # deletable msgpriority # timeout # hour clear

Sends a uninterpreted data message to the device. For a UtilityPRO thermostat, the text message is limited to 21 characters if the message priority is 7 and 63 characters is the message priority is below 7. Contact Cooper Power Systems Customer Service for more information.

extended tier # rate # cmd ‘cmd’ display # timeout # delay #

Sends tier pricing values to the thermostat.

utility info chan # [cost [cents]] [usage] “optional text”

Configures a thermostat channel to display cost and usage information. Cost values display as dollars by default, but the values can be displayed as cents by using the cents option. The “optional text” argument is used as a display name for the channel on the thermostat usage screen.

utility usage

channel:past usage:value,

channel:present usage:value,

channel:past cost:value,

channel:present cost:value

Sends usage and cost information to the thermostat.

serial #

Unique identifying serial number of the field device.

targetloadamps #

Specifies the ampere value for a field device. The resolution is 0.1 A.

 

The SPID address is allowed only if a serial number is being targeted. The feeder is a 16-bit bitmap, however it must be converted to a value between 0 and 65535 to be sent in a command. The Program and Splinter levels always require a Load specifier, since Program and Splinter addresses always target a Load. You can assign more than one Load by separating the parameters with commas, but the number of parameters in the Program and/or Splinter specifiers must match the number of Loads exactly.

Examples

putconfig xcom sync

putconfig xcom setstate relaymask 1 hold fan auto system off

putconfig xcom temp 72

putconfig xcom timeout 120

putconfig xcom timesync 06022003

putconfig template “A/C 1”

putconfig xcom assign s 250 g 101 b 2754 f 65535 z 55112 u 12 p 11,11,11 r 1,2,3 load 1,2,3

putconfig xcom assign p 21,100 load 1,3

putconfig xcom assign g 252

putconfig xcom service out temp

putconfig xcom schedule weekday 06:00, 68, 78, 10:00, 64, 74, 14:00, 68, 78, 16:00, 64, 74

putconfig xcom schedule sat 06:00, 70, 76, 10:00, 66, 76, 14:00, 70, 76, 16:00, 66, 76

putconfig xcom data 'Control likely today' port 0 deletable msgpriority 7 timeout 30 hour clear

putconfig xcom extended tier 2 rate 254 cmd 5 display 3 timeout 600 delay 5432

putconfig xcom utility info chan 1 cost usage "Channel 1"

putconfig xcom utility usage 1:past usage:1111, 1:present usage:2222,

1:past cost:3333, 1:present cost:4444

putconfig xcom serial 53452141 targetloadamps 20.1

 

EMETCON

putconfig EMETCON [address (gold x silver x bronze x lead meter x load x)] [address unique (address)] [alarm_mask (tamper | power_fail | under_voltage | over_voltage | pf_carryover | rtc_adjusted | holiday | dst_change | disconnect | read_corrupted)] [autoreconnect (disable | enable)] [centron [ratio N] [display (4x1|4x10|5x1) [test (1|7)] [errors]] [centron reading (forward | reverse)] [channel N (ied|2-wire|3-wire|none) [input N] [multiplier n.nn]] [disconnect [load limit (demand threshold kW) (connect delay minutes)] [cycle (disconnect minutes) (connect minutes)] [freeze day N] [group (enable|disable)] [holiday N mm/dd/yyyy [mm/dd/yyyy] [mm/dd/yyyy]] [ied (scan interval delay|class number offset)] [install] [install basic] [intervals|interval (li|lp)]  [multi_role number fn von vin stfn ] [onoffpeak|minmax] [multiplier kyz offset mult] [outage threshold N] [reading (forward|reverse)] [role rolenum fixb outb inb ] [tou (enable|disable)] [timesync] [timezone (et|ct|mt|pt)|(hours|-hours)]  [tou SMTWRFSH [schedule N [rate/hh:mm …] …] default rate]

 

The arguments are as follows:

 

Argument

Description

EMETCON

Target type of putconfig message.

address (gold x  silver x  bronze x  lead meter x  load x)

Uploads EMETCON addressing to a field device.

address unique (address)

Uploads a unique address to the field device.

alarm_mask (tamper | power_fail | under_voltage | over_voltage | pf_carryover | rtc_adjusted | holiday | dst_change | disconnect | read_corrupted)

Clears the status flag of the event that occurred.

autoreconnect (disable | enable)

Enables and disables the reconnect button on the MCT-410D disconnect collar. If the reconnect button is disabled, the ‘getconfig model’ command will return “reconnect button disabled” in the response.

centron [ratio N] [display (4x1|4x10|5x1) [test (1|7)] [errors]

Uploads the CENTRON multiplier (N) and display parameters to the MCT-410cL. If the display parameters are not specified, the display setup is reset to the following: 5x1 display (5 digits, 1 kWh resolution), no LCD segment test, LCD error display disabled.

centron reading (forward | reverse)

Uploads initial readings to the MCT-410cL.

channel N (ied|2-wire|3-wire|none) [input N] [multiplier n.nn]

Writes the channel configuration for the specified MCTs load profile channel. If no input is specified for a pulse input, the channel is used as the input number. The multiplier is ignored if the input is from an IED.

disconnect [load limit (demand threshold kW) (connect delay minutes)] [cycle (disconnect minutes) (connect minutes)]

Uploads the disconnect address from the database. This command can also upload ‘load limit disconnect parameters’ and ‘cycle parameters’. If the ‘load limit parameters’ are not specified, the defaults are:

demand threshold kW: 0 (disabled) and connect delay: 5 (minutes)

If the ‘cycle parameters’ are not specified, they are unchanged. If either of the ‘cycle parameters’ is specified as zero, cycling mode is disabled.

freeze day N

Sets the meter to freeze on a specific day of the month.  Where N = 0 through 31; use 0 to disable the freeze option.

group (enable|disable)

Enables or disables group addressing.

holiday N mm/dd/yyyy [mm/dd/yyyy] [mm/dd/yyyy]

Writes the holiday dates to the field device, starting with holiday N. Up to three holidays can be entered in the device at one time.

ied class number offset

Sets class and offset of an IED.

ied scan interval delay

Changes the interval and delay of a demand reading for an IED.

install

Verifies the device SPEC and uploads LI and LP intervals.

install basic

Uploads the device configuration to an MCT-470 or MCT-430.

intervals

Uploads last interval demand/voltage and load/voltage profile settings to a field device.

interval (li|lp)

Uploads demand interval information to a field device. (li = last interval demand, lp = load profile)

minmax

Configure the field device to record minimum and maximum values.

multiplier kyz offset mult

Uploads an internal multiplier into a field device.

multi_role count fn von vin stfn

Uploads multiple role configurations data to a repeater device. Contact Cooper Power Systems Customer Service for more information.

 

onnoffpeak

Configures the field device to record on peak and off peak values.

outage threshold N

Configures the minimum outage cycles before an outage is recorded. A value of 0 disables the outage threshold. Valid settings for N are 0 and 15 through 60 cycles.

role rolenum fixb outb inb stages

Uploads a role configuration data to a repeater device. Contact Cooper Power Systems Customer Service for more information.

timesync

Sends a time synchronization command to the field device.

timezone (et|ct|mt|pt)

Uploads the specified timezone to the field device.

timezone (hours|-hours)

Uploads the timezone to the field device as a raw hour offset.

tou (enable|disable)

Enables or disables TOU on a meter. When enabled, readings are stored in the rates specified by the active TOU schedule. When disabled, all readings are accumulated in the default TOU rate.

[tou SMTWRFSH [schedule N [rate/hh:mm …] …] default rate]

Writes a TOU schedule to an MCT. The first parameter is a table of days, which represents which schedule is used on which day. An MCT has 4 schedules that can be used. The schedules themselves are specified by a schedule number and a sequence of rate changes.

putconfig EMETCON centron display errors

putconfig EMETCON channel 2 none

putconfig EMETCON disconnect load limit 13.4 0

putconfig EMETCON disconnect cycle 14 12

putconfig EMETCON disconnect load limit 11 2 cycle 15 19

putconfig EMETCON freeze day 2

putconfig EMETCON group enable

putconfig EMETCON holiday 3 07/04/2006

putconfig EMETCON holiday 1 9/2/2006 11/25/2006 12/24/2006

putconfig EMETCON ied scan 30 5

putconfig EMETCON ied class 72 02

putconfig EMETCON install

VERSACOM

putconfig [VERSACOM] [template name] [utility # |aux # |section # |class (#,..,# | 0x##) |division (#,..,# | 0x##)] [fromutility # utility #| fromsect # section #| fromclass # class #| fromdivsion # division #] [assign [u#] [a#] [s#] [c0x##] [d0x##]] [service (in|out) [temp]] [led (y | n)(y | n)(y | n)] [reset [rtc] [lf] [r1] [r2] [r3] [r4] [cl] [pt]] [prop time] [cold [r1 time] [r2 time] [r3 time]] [cycle [r1 count] [r2 count] [r3 count]] [scram [r1 time] [r2 time] [r3 time]] [vdata # # # # # #] [raw # # # # # #]

 

The meanings of the arguments are as follows:

 

Argument

Description

VERSACOM

Target type of putconfig message. The target type defaults to VERSACOM.

template name

Configures a device to use the addressing of the specified Load Group. Double quotes are required if the name of the load group contains spaces.

utility #

Assigns an utility id number to the device.

aux #

Assigns an auxiliary utility id number  to the device.

section #

Assigns a section number  to the device.

class #,..,#

Assigns class numbers (up to 16 numbers) to the device.

class 0x##    (2 bytes)

Assigns class numbers to the device.
# is a hexadecimal number in the spec. format.

This format enumerates bits 1-16 with the MSB as 1.

Example 8000 = Class 1, F000 Class 1-4.

division #,..,#

Assigns a division numbers (up to 16 numbers) to the device.

division 0x##    (2 bytes)

Assigns a division numbers to the device.
# is a hexadecimal number in the correct format.

This format enumerates bits 1-16 with the MSB as 1.

Example 8000 = Class 1, F000 Class 1-4.

fromutility # utility #

Reassigns utility addressing using the specified utility ID

fromsect # section #

Reassigns section addressing using the specified section ID

fromclass # class #

Reassigns class addressing using the specified class ID

fromdivision # division #

Reassigns division addressing using the specified division ID

assign [u#] [a#] [s#] [c0x##] [d0x##]

Assigns utility, auxiliary, section, class, and division ID.

 - [u#] = utility

 - [a#] = auxiliary

 - [s#] = section

 - [c0x##] = class

 - [d0x##] = division

 

Class and division addresses are a 2 byte quantity properly formatted per the protocol spec. This format enumerates bits 1-16 with the MSB as 1.

 

Example 8000 = Class 1, F000 = Class 1-4.

Any or all may be specified in one command and order is irrelevant.

service (in|out) [temp]

Place device in or out of service. (t is an optional temporary flag.)

led (y|n)(y|n)(y|n)

Configures a receiver's LEDs. (y = enable.)

LEDs are Load, Status, and Report respectively.

reset

Resets internal counters.

rtc

Resets the real-time clock

lf

Resets the LUF counter

rn

Resets relay n load control counter.

cl

Resets communications loss counter.

pt

Resets propagation test counter.

prop time

Sets propagation display time in hours. When a device receives the prop command, the LED is lit for the time specified.

Valid ranges vary, but 1-9 hours are acceptable for all devices.

cold [r1 time] [r2 time] [r3 time]

Sets the cold load pickup time for one or more relays on the VERSACOM device. Any or all may be specified in one command.

cycle [r1 count] [r2 count] [r3 count]

Sets the number of load control cycles for one or more relays on the VERSACOM device. Any or all may be specified in one command.

scram [r1 time] [r2 time] [r3 time]

Sets the scram enable/restore time for one or more relays on the VERSACOM device. Any or all may be specified in one command.

vdata # # # # # #

Sends an uninterpreted data command to the device. Contact Cooper Power Systems Customer Service for more information.

raw # # # # # #

Sends an uninterpreted configuration command to the device. Contact Cooper Power Systems Customer Service for more information.

Examples

putconfig template "A/C 1"

putconfig utility 10

putconfig fromutility 5 utility 1

putconfig assign u12 s8

putconfig service out temp

putconfig led y y n

putconfig reset rtc lf r1 r2 r3 cl pt

putconfig prop 2

putconfig cold r1 60

putconfig cycle r2 5

putconfig scram r3 30
