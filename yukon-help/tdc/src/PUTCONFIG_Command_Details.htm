PUTCONFIG Command Details

This command is used to send configuration type data to the various supported field devices. The command will default to the most basic or common configuration data for each device type. Most field device types support multiple pieces of configuration data. To allow for the programming of the different configuration data the PUTCONFIG command supports additional arguments to specify the other pieces of data.

VERSACOM

putconfig [VERSACOM] [template name] [utility # |aux # |section # |class (#,..,# | 0x##) |division (#,..,# | 0x##)] [fromutility # utility #| fromsect # section #| fromclass # class #| fromdivsion # division #] [assign [u#] [a#] [s#] [c0x##] [d0x##]] [service (in|out) [temp]] [led (y | n)(y | n)(y | n)] [reset [rtc] [lf] [r1] [r2] [r3] [r4] [cl] [pt]] [prop time] [cold [r1 time] [r2 time] [r3 time]] [cycle [r1 count] [r2 count] [r3 count]] [scram [r1 time] [r2 time] [r3 time]] [vdata # # # # # #] [raw # # # # # #]

 

The arguments are as follows:

 

Argument

Description

VERSACOM

Target type of putconfig message. The target type defaults to VERSACOM.

template name

Configures a device to use the addressing of the specified Load Group.

utility #

Assigns an utility id number to the device.

aux #

Assigns an auxiliary utility id number  to the device.

section #

Assigns a section number  to the device.

class #,..,#

Assigns class numbers (up to 16 numbers) to the device.

class 0x##    (2 bytes)

Assigns class numbers to the device.
# is a hexadecimal number in the spec. format.

This format enumerates bits 1-16 with the MSB as 1.

Example 8000 = Class 1, F000 Class 1-4.

division #,..,#

Assigns a division numbers (up to 16 numbers) to the device.

division 0x##    (2 bytes)

Assigns a division numbers to the device.
# is a hexadecimal number in the correct format.

This format enumerates bits 1-16 with the MSB as 1.

Example 8000 = Class 1, F000 Class 1-4.

fromutility # utility #

Reassigns utility addressing using the specified utility ID

fromsect # section #

Reassigns section addressing using the specified section ID

fromclass # class #

Reassigns class addressing using the specified class ID

fromdivision # division #

Reassigns division addressing using the specified division ID

assign [u#] [a#] [s#] [c0x##] [d0x##]

Assigns utility, auxiliary, section, class, and division ID.

 - [u#] = utility

 - [a#] = auxiliary

 - [s#] = section

 - [c0x##] = class

 - [d0x##] = division

 

Class and division addresses are a 2 byte quantity properly formatted per the protocol spec. This format enumerates bits 1-16 with the MSB as 1.

 

Example 8000 = Class 1, F000 = Class 1-4.

Any or all may be specified in one command and order is irrelevant.

service (in|out) [temp]

Place device in or out of service. (t is an optional temporary flag.)

led (y|n)(y|n)(y|n)

Configures a receiver's LEDs. (y = enable.)

LEDs are Load, Status, and Report respectively.

reset

Resets internal counters.

rtc

Resets the real-time clock

lf

Resets the LUF counter

rn

Resets relay n load control counter.

cl

Resets communications loss counter.

pt

Resets propagation test counter.

prop time

Sets propagation display time in hours. When a device receives the prop command, the LED is lit for the time specified.

Valid ranges vary, but 1-9 hours are acceptable for all devices.

cold [r1 time] [r2 time] [r3 time]

Sets the cold load pickup time for one or more relays on the VERSACOM device. Any or all may be specified in one command.

cycle [r1 count] [r2 count] [r3 count]

Sets the number of load control cycles for one or more relays on the VERSACOM device. Any or all may be specified in one command.

scram [r1 time] [r2 time] [r3 time]

Sets the scram enable/restore time for one or more relays on the VERSACOM device. Any or all may be specified in one command.

vdata # # # # # #

Sends an uninterpreted data command to the device. Contact Cannon Technical Support for more information.

raw # # # # # #

Sends an uninterpreted configuration command to the device. Contact Cannon Technical Support for more information.

Examples

putconfig template "A/C 1"

putconfig utility 10

putconfig fromutility 5 utility 1

putconfig assign u12 s8

putconfig service out temp

putconfig led y y n

putconfig reset rtc lf r1 r2 r3 cl pt

putconfig prop 2

putconfig cold r1 60

putconfig cycle r2 5

putconfig scram r3 30

 

ExpressCom

putconfig xcom [sync] [setstate [relaymask #] [run] [hold] [fan (on|off|auto)] [system (off|heat|cool|emheat)] [temp value] [timeout time]] [timesync date hh:mm] [template name] [raw #] [data #] [maintenance #] [assign|address [spid #] [geo #] [sub #] [feeder #] [zip #] [uda #] [program #] [splinter #]] [relay|load #] [service (in|out|enable|disable) [t] (noclp|noled|bitP|bitL|cancel| offhours time)] [schedule day HH:MM, Heat Temp, Cool Temp, ...]

 

The arguments are as follows:

 

Argument

Description

xcom

Target type of putconfig message.

sync

Keeps devices locked to a paging company and cap code.

setstate

Sets the thermostat state.

relaymask

Specifies the relays of the device.

run

Runs a program

hold

Holds the current temperature.

fan (on|off|auto)

Sets the fan state to on, off, or auto.

system (off|heat|cool|emheat)

Sets the system state to off, heat, cool, or emheat (heat pump).

 

temp value

Sets the temperature.

timeout time

Sets the length of time the program runs until it times out.

timesync date hh:mm

Sends a time sync with the current time of day. Including a date sends a the current time and date. Specifying the hours and minutes sends that particular time.

template name

Configures a device to use the addressing of the specified Load Group.

raw #

Sends an uninterpreted configuration command to the device. Contact Cannon Technical Support for more information.

data #

Sends an uninterpreted data message to the device. Contact Cannon Technical Support for more information.

maintenance #

Sends an uninterpreted maintenance function to the device. Contact Cannon Technical Support for more information.

assign [s #] [g #] [b #] [f #] [z #] [u #] { [p #] [r #] [load #] }

Assigns a new ExpressCom address to the device. The address must be entered in the order listed.

s = Spid

g = Geographic

b = suBstation

f = Feeder

z = Zip code

u = User-defined

p = Program

r = splinteR

service
(in|out|enable|disable) [temp] (noclp|noled|offhours time)

Sends a command to take the device out of service (disable) or to place the device in service (enable). The temp option makes the service command a temporary command. The noclp option keeps the device from running cold load pickup during startup. The noled option turns of the LEDs when the device is out of service. The offhours time option takes the device out of service for a specified number of hours.

schedule day HH:MM, Heat Temp, Cool Temp, ...

Assigns period schedules to a thermostat.

 

Examples

putconfig xcom sync

putconfig xcom setstate relaymask 1 hold fan auto system off

putconfig xcom temp 72

putconfig xcom timeout 120

putconfig xcom timesync 06022003

putconfig template A/C 1

putconfig xcom assign s 250 g 101 b 2754 f 65535 z 55112 u 12 p 11,11,11 r 1,2,3 load 1,2,3

putconfig xcom assign p 21,100 load 1,3

putconfig xcom assign g 252

putconfig xcom service out temp

putconfig xcom schedule weekday 06:00, 68, 78, 10:00, 64, 74, 14:00, 68, 78, 16:00, 64, 74

putconfig xcom schedule sat 06:00, 70, 76, 10:00, 66, 76, 14:00, 70, 76, 16:00, 66, 76

 

 

EMETCON

putconfig EMETCON [timesync] [intervals|interval (li|lp)] [onoffpeak|minmax] [disconnect|connect] [multiplier kyz offset mult] [role number fixb outb inb] [multi_role number fn von vin stfn ][ied (scan interval delay|class number offset)] [install] [group (enable|disable)]

 

The arguments are as follows:

 

Argument

Description

EMETCON

Target type of putconfig message.

timesync

Forces a time sync

intervals

Uploads last interval demand/voltage and load/voltage profile settings to a field device.

interval (li|lp)

Uploads demand interval information to a field device. (li = last interval demand, lp = load profile)

onnoffpeak

Configures the field device to record on peak and off peak values.

minmax

Configure the field device to record minimum and maximum values.

disconnect

Sends a disconnect command to a disconnect meter base.

connect

Sends a connect command to a disconnect meter base.

multiplier kyz offset mult

Uploads an internal multiplier into a field device.

role rolenum fixb outb inb stages

Uploads a role configuration data to a repeater device. Contact Cannon Technical Support for more information.

multi_role count fn von vin stfn

Uploads multiple role configurations data to a repeater device. Contact Cannon Technical Support for more information.

raw [start=0xhexnum 0xbyte1 0xbyte2]

Uploads a raw block of data to a field device. Contact Cannon Technical Support for more information.

ied scan interval delay

Changes the interval and delay of a demand reading for an IED.

ied class number offset

Sets class and offset of an IED.

install

Verifies the SPEC and uploads LI and LP intervals

group (enable|disable)

Enables or disables group addressing

address (gold x silver x bronze x lead meter x load x)

Uploads EMETCON addressing to a field device.

 

Examples

putconfig EMETCON timesync

putconfig EMETCON intervals

putconfig EMETCON interval li

putconfig EMETCON onoffpeak

putconfig EMETCON minmax

putconfig EMETCON connect

putconfig EMETCON disconnect

putconfig EMETCON multiplier kyz 1 1.25

putconfig EMETCON role 12 7 6 0 0

putconfig EMETCON ied scan 30 5

putconfig EMETCON ied class 72 02

putconfig EMETCON install

putconfig EMETCON group enable

putconfig EMETCON address gold 2 silver 5

putconfig EMETCON address lead meter 3 load 6
