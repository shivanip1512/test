CONTROL Command Details

This command is used to send control commands to the various supported field devices. To allow for different types of control, the CONTROL command supports the following arguments.

 

control [xcom] [cycle percent (period time) (count #) relay #] [delay (time # | until hh:mm) ] [noramp] [priority #] [rand start time] [rand stop time] [restore|terminate] [shed time relay #] [open|close|connect|disconnect]

 

control [xcom] [celsius] [delta] [mode (heat|cool|both)] [delay (time # | until hh:mm)] [noramp] [priority #] [rand start time] [rand stop time] [setpoint (min temp) (max temp) (tr time) (ta time) (tb time dsb temp) (tc time) (td time dsd temp) (te time) (tf time dsf temp)] [tcycle load # (period time) (count #) (ctrl temp #) (limit temp afallback percent) (maxrate rate bfallback percent)]

 

The standard arguments are as follows:

 

Argument

Description

xcom

Sends the command using the ExpressCom protocol.

priority #

Priority of the ExpressCom message. A message with a lower priority (9) is ignored if a message with a higher priority (0) has already been received by the device.

cycle percent

Percent of the time period that the device is controlled.

 

period time

Optional length of the cycle period. Defaults to 30 minutes if omitted.

 

count #

Optional number of periods to operate before stopping. Defaults to 8 counts if omitted.

 

shed time

Defines a shed command and the length of the shed.

 

relay #

The relay of the device to which the command is being sent.

delay time #

Execution of this command is delayed by the specified number minutes.

delay until hh:mm

Execution of this command is delayed until the time hh:mm, where hh:mm is in the future and in 24 hour time format.

noramp

Causes the device to shed immediately and adds the randomization time to the end of the first off cycle.

rand start time

Adds a random start time to the beginning of the shed control time.

rand stop time

Adds a random stop time to the end of the shed control time.

restore

Restores a shed command. Ends a cycle with a randomization of 2.5 minutes.

terminate

Terminates a cycle command.

open

Sends a command to a field device to open the relay that is used as the open control.

close

Sends a command to a field device to close the relay that is used as the close control.

connect

Sends a connect command to an MCT.

disconnect

Sends an open command to an MCT.

 

The thermostat command arguments are as follows:

 

Argument

Description

xcom

Sends the command using the ExpressCom protocol.

priority #

Priority of the ExpressCom message. A message with a lower priority (9) is ignored if a message with a higher priority (0) has already been received by the device.

celsius

All temperatures referenced in the thermostat control command are Celsius. Default is Fahrenheit.

delay time #

Execution of this command is delayed by the specified number minutes.

delay until hh:mm

Execution of this command is delayed until the time hh:mm, where hh:mm is in the future and in 24 hour time format.

delta

Temperatures referenced in the thermostat control command will be positive and negative offsets from the program setpoint. Default is absolute values.

mode (heat|coo|both)

Selects the system heating or cooling, which is to be targeted in a thermostat control command

noramp

Causes the device to shed immediately and adds the randomization time to the end of the first off cycle.

rand start time

Adds a random start time to the beginning of the shed control time.

rand stop time

Adds a random stop time to the end of the shed control time.

setpoint

Defines a thermostat setpoint command.

 

min temp

Optional absolute temperature below which the stat will not go.

 

max temp

Optional absolute temperature above which the stat will not go.

 

tr time

Optional random time period (0 - time) minutes added to the setpoint command.

 

ta time

Optional time in minutes prior to the setpoint command execution.

 

tb time

Optional time in minutes of ramp transition to the first temperature waypoint. Requires the option dsb to be defined.

 

dsb temp

Optional value in degrees indicating the final temperature at the end of tb time period. May be absolute or delta depending upon delta parameter.

 

tc time

Optional time in minutes to hold at the temperature set in the tb/dsb stage of the control.

 

td time

Optional time in minutes of ramp transition to the control temperature waypoint. Requires the option dsd to be defined.

 

dsd temp

Optional value in degrees indicating the final temperature at the end of td time period. May be absolute or delta depending upon delta parameter.

 

te time

Optional time in minutes to hold at the temperature set in the td/dsd stage of the control.

 

tf time

Optional time in minutes of ramp transition to the final temperature waypoint. Requires the option dsf to be defined.

 

dsf temp

Optional value in degrees indicating the final temperature at the end of tf time period. May be absolute or delta depending upon delta parameter.

tcycle

Defines a thermostat cycle command.

 

 

load #

The load number to control. This is a required parameter and it is normally 1, indicating the primary heating or cooling load.

 

period time

Optional length of the cycle period. Defaults to 30 minutes if omitted.

 

count #

Optional number of periods to operate before stopping. Defaults to 8 counts if omitted.

 

ctrl temp #

Optional temperature setpoint during the portion of the period in which the system is allowed to operate.

 

limit temp
afallback percent

Optional limit temperature. This parameter limits the maximum or minimum temperature allowed based upon a threshold of 72 F. If the value entered is greater than 72, it is considered a maximum if less, a minimum.

The afallback percent argument is an optional fallback control percentage. If the limit temperature is reached, the control percentage will change to the afallback percentage.

 

 

maxrate rate  
bfallback percent

Optional maximum temperature change rate per hour. May be a positive or negative integer.

The bfallback percent argument is an optional fallback control percentage. If the maxrate is exceeded, the control percentage changes to the bfallback percentage.

Examples

Examples of a standard command are:

control xcom cycle 50 period 30 count 6 delay time 60 relay 1 priority 0

control xcom shed 120 rand start 45 rand stop 60 delay until 10:00 relay 3

control xcom restore

control xcom terminate

control xcom open

control xcom close

control xcom connect

control xcom disconnect

 

Examples of a thermostat command are:

control xcom delta mode both tcycle 50 load 1 period 60 count 4 maxrate -5 bfallback 25

control xcom setpoint delta mode cool min 68 tr 30 ta 30 tb 120 dsb -5 tc 60 td 240 dsd 10 te 60

control xcom setpoint min 68 tb 120 dsb 68 tc 60 td 240 dsd 78 te 60

control xcom tcycle 50 period 30 count 8 delay time 30 load 1 priority 9
