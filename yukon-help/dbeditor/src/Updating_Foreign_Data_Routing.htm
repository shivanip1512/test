Updating a Foreign Data Routing Entry

To update foreign data routing:

 

  1. Select the Foreign Data Routing item to update from the Foreign Data Routing list.

  2. A different interface can be selected from the Interface drop-down list. The available interfaces are:  ACS, ACSMULTI, BEPC,  CYGNET, DSM2FILEIN, DSM2IMPORT, INET, LIVEDATA, LODESTAR_ENH, LOADSTAR_STD, OPC, PI, RCCS, RDEX, STEC, SYSTEM, TELGYR, TEXTEXPORT, TEXTIMPORT, TRISTATE, TRISTATESUB, VALMET, WABASH, and XA21LM.  The Data Parameters change depending on the type of interface you select.

  3. A different data direction can be selected from the Direction drop-down list. This controls whether the data is received or sent.

  4. Depending on the Interface selected, different options appear under Device Parameters:

    • If you selected ACS, select Real or Pseudo from the Category drop-down list, and enter the remote name in the Remote text box and the point name to import or export in the Point text box.

    • If you selected ACSMULTI, select Real or Pseudo from the Category drop-down list, enter the remote name in the Remote text box, enter the point name to import or export in the Point text box, and enter a destination or source in the Destination/Source text box.

    • If you selected CYGNET or TEXTEXPORT, enter the point ID to import or export in the PointID text box.

    • If you selected DSM2FILEIN, enter the option in the Option Number text box and the point ID to import in the Point ID text box.

    • If you selected DSM2IMPORT or VALMET, enter the point name to import or export in the Point text box.

    • If you selected INET or RCCS, enter the device name in the Device text box, the point name to import or export in the Point text box, and the destination/source in the Destination/Source text box.

    • If you selected LIVEDATA, enter the address in the Address text box and select the type of data from the Data Type drop-down list.

    • If you selected LODESTAR_ENH or LOADSTAR_STD, enter the customer in the Customer text box, the channel in the Channel text box, the drive path in the DrivePath text box, and the filename in the Filename text box.

    • If you selected OPC, enter the name of the OPC server in the Server Name text box, enter the name of the OPC Group in the OPC Group text box, and enter the name of the OPC Item in the OPC Item text box.

    • If you selected PI, enter the tag name of the point in the Tag Name text box. If the polling method is being used, enter how often to poll the PI database in the Period (sec) text box. If the notification method is being used, do not enter a value in the Period (sec) text box.

    • If you selected RDEX, enter the point name in the Translation text box and the destination/source IP address in the Destination/Source text box.

    • If you selected STEC, TRISTATE, or TRISTATESUB select the point to import from the Point list.

    • If you selected SYSTEM, enter the client name in the Client text box.

    • If you selected TELEGYR, enter the point name to import in the Point text box and select a group from the Group drop-down list.

    • If you selected TEXTIMPORT, enter the point ID to import in the PointID text box, enter the location to save the file in the DrivePath text box and enter the name of the file in the FileName text box.

    • If you selected WABASH, enter the name of the schedule in SchedName text box, enter the location to save the control file to in the Path text box (c:\Yukon\Server\Export\), and enter the name of the control file in the File Name text box (control.txt).

    • If you selected XA21LM, enter the point number of the point on the XA/21 system to import or export in the Translation text box.

  5. Click the Update button to update the foreign data routing.
