Comm Channel Timing Tab

The following changes can be made to the timing settings of the communication channel:

 

  1. The number of milliseconds to wait can be set in the following spin boxes:

    • Pre Tx Wait – Defines the number of milliseconds to wait between completing the last transmission and starting the next transmission.

    • RTS To Tx Wait – Defines the number of milliseconds to wait between asserting Request to Send (RTS) and sending data to the port.

    • Post Tx Wait – Defines the number of milliseconds to wait after transmission before returning RTS to normal.

    • Receive Data Wait – Defines the millisecond delay used to let the channel switch directions before flushing the receiver buffer and looking for the received data.

  2. The number of time-out seconds can be set in the Additional Time Out spin box. These additional time-out seconds are added to the calculated time-out.
