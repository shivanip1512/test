Load Management

The Load Management database allows you to create control areas, load programs and load groups, and points.

 

Build a new system database in the following order:

  1. Define the load groups. A load group defines the communication protocol and the device addressing needed to control field devices.

  2. Optionally, define program constraints for load programs.

  3. Define the load programs. Load programs define the loads to control and the method of control. The three types of load programs are:

    • Direct Control - sends controls to the field hardware to reduce load. Load groups are assigned to direct control programs.

    • Curtailment- requires that the customer acknowledge the curtailment notification. Customers are assigned to curtailment programs.

    • Energy Exchange - provides customer with the opportunity to shed load for energy credits. Customers are assigned to energy exchange programs.

  4. Define the control areas. Control areas define geographical regions used for controlling loads to reduce peak. A Control Area may be the entire system, portion of the system, substation, or any defined controllable area. Two trigger points can be defined per control area.

Note: When two control triggers are defined, you must select whether both are required active before control begins for the control area.

  1. Optionally, create control scenarios. A Control Scenario defines when to start load programs and the gears to use for the programs.

  2. Define the points for load groups. A point is required for every discrete piece of information that is collected. The two types of load management points are:

    • Analog - an analog point reads a value directly from the transducer or electronic device.

    • Status - a status point reads the value of a contact, usually open or closed. An example of a status point might be the relay on a verification load control receiver.
