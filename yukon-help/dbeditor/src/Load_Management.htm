Load Management

The Load Management database allows control areas, load programs and load groups, and points to be defined in the Yukon system.

 

Cooper Power Systems recommends creating a new Load Management database in the following order:

  1. Define the load groups in the Yukon system. A load group defines the communication protocol and the device addressing needed to control field devices.

  2. Define the load programs in the Yukon system. A load program defines the load groups to control and the method of control.

  3. Optionally, define program constraints for load programs in the Yukon system.

  4. Define the control areas in the Yukon system. A control area defines a geographical region used for controlling loads to reduce peak. A control area may be the entire system, portion of the system, substation, or any defined controllable area. Two trigger points can be defined per control area.

Note: When two control triggers are defined, select whether or not both triggers are required to be active before control can start for the control area.

  1. Optionally, create control scenarios in the Yukon system. A control scenario defines when to start load programs and the gears to use for the programs.

  2. Define the points for load groups. A point is required for every discrete piece of information that is collected. The two types of Load Management points are:

    • Analog – An analog point reads a value directly from the transducer or electronic device.

    • Status – A status point reads the value of a contact, usually open or closed. An example of a status point might be the relay on a verification load control receiver.
