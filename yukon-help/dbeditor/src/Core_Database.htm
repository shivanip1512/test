Core Database

The Core Database contains all of the information necessary for the Yukon system to communicate with field devices. This includes information about the comm channels, radio or phone lines, baud rates, device addresses, data collection, point definitions, and information on how to handle data returned as well as alarms and errors.

 

Cooper Power Systems recommends creating a new core database in the following order:

 

  1. Create the communication channels for the Yukon system. Communication channels (comm channels) exchange data with field devices.

  2. Create the devices that use comm channels to communicate. A device is the physical field hardware used by the Yukon software to gather data. This includes Power Line Carrier (PLC) infrastructure hardware, data gathering devices, and control devices.

 

The following device types, which use comm channels to communicate, can be created in the Database Editor:

    • Signal Transmitter

    • Electronic Meter

    • RFMESH (Radio Frequency Node)

    • RTU

    • Virtual Device

    • Grid Advisor

Note: Metering Control Transponders (MCT), repeaters, and two-way Load Control Receivers (LCR) cannot be created until communication routes exist.

 

  1. Create the communication routes for the Yukon system. A route is a communication path to a field device. These paths are used by the Yukon system to control the number of communications distributed on the system. Macro routes combine multiple individual routes into a combination route and send messages to multiple areas of a system.

 

When a signal transmitter is added to the Yukon Editor, the transmitter communication route is automatically added to the database. Examples of signal transmitters include:

    • CCU

    • LCU

    • Paging Tap Terminal

    • RTC

    • RTU-LMI

    • SNPP Terminal

    • TNPP Terminal

    • TCU

    • WCTP Terminal

 

  1. Create any repeaters through which the downline communication travels to field devices. A repeater is needed when a CCU is unable to communicate with an entire region. Routes defined using repeaters originate from the transmitter and have their signal rebroadcast by the repeater.

  2. Create the MCT devices and then assign names, addresses, and routes.

  3. Create additional points on the devices. A point is required for every discrete piece of information that is collected.

The four types of points are:

    • Analog – Reads the value directly from the transducer or electronic device. Examples include Voltage or Amperage transducer on an RTU or a pseudo point.

    • Status – Reads the value of the contact or digital input, usually open or closed. Examples a switch status (open or closed) or status point residing on a foreign system. A status point can also have control offset.

    • Accumulator – Measures any quantity that is a simple count accumulation, such as the number of pulses. Examples include kWh energy point or kW demand (energy over time).

    • Calculated – Calculates values from math operations performed on other points.

  1. Create state groups. State groups contain message text, color-codes, and images which represent the possible states of a status point.
