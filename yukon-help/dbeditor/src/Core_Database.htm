Core Database

The Yukon Core Database contains all of the information necessary for the Master Station computer to communicate with field devices. This includes information about the comm channels, radio or phone lines, baud rates, device addresses, data collection, point definitions, and information on how to handle data returned as well as alarms and errors.

 

Cannon Technologies recommends creating a new core database in the following order:

 

  1. Define the communication channels on the system. Communication channels (comm channels) exchange data with field devices. Any number of comm channels can be used with the system.

  2. Define the devices that are attached to the comm channels. A device is the physical field hardware used by the Yukon Master Station software to gather data. This includes power line carrier infrastructure hardware, data gathering devices, and control devices.

    The following devices can be created in the Yukon Editor.

    • Signal Transmitters

    • Electronic Meters

    • Remote Terminal Units (RTU)

    • Virtual Devices

Note: MCTs and repeaters cannot be defined until routes are defined.

 

  1. Define the communication routes. A route is a communication path through the distribution system to a field device. These paths are used by the system to control the number of communications distributed on the system. Macro routes combine multiple individual routes into a combination route and send load management messages to multiple areas of a system.

    When a signal transmitter is added to the Yukon Editor, the transmitter route is automatically added to the database. Examples of signal transmitters include:

    • CCU

    • LCU

    • Paging Tap Terminal

    • RTC

    • RTU-LMI

    • SNPP Terminal

    • TCU

    • WCTP Terminal

 

  1. Define any repeaters through which the downline communication is going to field devices. A signal repeater route is needed when the communication route is unable to reach its entire region. Routes defined using repeaters originate from the transmitter and have their signal rebroadcast by the repeater.

  2. Define the Metering and Control Transponders (MCT) and then assign names, addresses, and routes.

  3. Define points on the devices. A point is required for every discrete piece of information that is collected.

The four types of points are:

    • Analog - reads the value directly from the transducer or electronic device. Examples include Voltage or Amperage transducer on an RTU or a pseudo point.

    • Status - reads the value of the contact or digital input, usually open or closed. Examples a switch status (open or closed) or status point residing on a foreign system. A status point can also have control..

    • Accumulator - measures any quantity that is a simple count accumulation, such as the number of pulses. Examples include kWh energy point or kW demand (energy over time).

    • Calculated - derives values from math operations performed on other points.

  1. Define state groups. State groups contain message text, color-codes, and images which represent the possible states of a status point.
