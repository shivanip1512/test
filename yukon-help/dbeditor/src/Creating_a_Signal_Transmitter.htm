Creating a Signal Transmitter

Perform the following steps to create a signal transmitter:

 

  1. From the View menu, select Core.

  2. From the Create menu, select Device.

  3. Select Signal Transmitter from the list on the left.

  4. Select the type of signal transmitter to create from the list on the right:

    • CCU-710A – Carrier Control Unit.

    • CCU-711 – Carrier Control Unit.

    • CCU-721 – Carrier Control Unit.

    • LCU-EastRiver – Local Control Unit.

    • Paging Tap Terminal – Telocator Alphanumeric Protocol.

    • RTC – Modbus RTU Transmitter.

    • RTU-LMI – Modbus RTU Transmitter.

    • SNPP Terminal – Simple Network Paging Protocol.

    • TNPP Terminal – Telocator Network Paging Protocol.

    • TCU-5000 – VHF Transmitter Control Unit.

    • TCU-5500 – VHF Transmitter Control Unit.

    • WCTP Terminal – Wireless Communication Transfer Protocol.

  5. Click the Next button.

  6. Enter a name for the signal transmitter in the Device Name text box.

  7. If a CCU-710A, CCU-711, LCU, RTC, or TCU was selected as the signal transmitter, enter an address in the Physical Address text box.

    • For the CCU-710A and CCU-711, the physical address is located on the power supply dial inside the CCU enclosure. If the CCU power supply does not have a dial setting, the physical address is zero.

    • For Local Control Units (LCU) and Transmitter Control Units (TCU), the physical address is programmed into the signal transmitter.

If a CCU-721 was selected as the signal transmitter, enter the address in the Slave Address text box. The physical address of a CCU-721 is accessible from the front panel display.

 

If a Paging Tap Terminal was selected as the signal transmitter:

    1. Enter a pager number in the Pager Number text box.

    2. If a password is required to log into to the paging account, select the Password Required check box and enter the password of the paging account in the Password text box.

If an SNPP Terminal was selected as the signal transmitter:

    1. Enter a pager number in the Pager Number text box.

    2. Enter the login of the SNPP account in the Login text box.

    3. Enter the password of the SNPP account in the Password text box.

If a TNPP Terminal was selected as the signal transmitter and a password is required to log into the TNPP account, select the Password Required check box and enter the password of the TNPP account in the Password text box.

 

If a WCTP Terminal was selected as the signal transmitter:

  1. Enter the pager number in the Pager Number text box.

  2. Enter the sender ID/address of the WCTP account in the Sender text box.

  3. Enter the password of the WCTP account in the Password text box.

  4. Enter the security code of the WCTP account in the Security Code text box.

  5. Click the Next button.

    • If a CCU, LCU, RTC, or RTU-LMI, or TCU was selected as the signal transmitter, skip to step 6.

    • If a Paging Tap Terminal, SNPP Terminal, or WCTP Terminal was selected as a signal transmitter, skip to step 9.

    • If a TNPP Terminal was selected as the signal transmitter, complete the following steps:

      1. Enter the source address in the Origin Address text box. The source address indicates the node from which the packet originated.

      2. Enter the destination address in the Destination Address text box. The destination address indicates the node that processes the transactions in the packet.

      3. Enter the inertia value in the Inertia text box. The inertia value is the maximum number of nodes that the packet is allowed to pass through before it reaches the destination.

      4. Select the type of protocol used for sending pages the Protocol drop-down list.

      5. Select the format of the data used for sending pages from the Data Format drop-down list.

      6. Select the type of format used for sending pages from the Identifier Format drop-down list. The CAP page format contains all the raw information for sending a message to a pager. The ID page format only contains the pager's PIN and it must be translated at a paging terminal.

      7. Enter the pager ID of the outgoing pages in the Pager ID text box.

      8. Enter the RF channel designator in the Channel text box.

      9. Enter the function code in the Function Code text box.

      10. Enter the RF zone designator in the Zone text box.

      11. Click the Next button and skip to step 9.

  6. Select the check boxes of the scan categories to enable. The scan categories vary depending on the type of signal transmitter selected in step 4.

    • If the signal transmitter is an RTU-LMI, the following scan category is displayed:

      • Integrity Rate – Collects all point values from the signal transmitter

    • If the signal transmitter is an LCU, the following scan categories are displayed:

      • Integrity Rate – Collects all point values from the signal transmitter

      • Report by Exception – Reports if anything has changed on the signal transmitter

      • Accumulator Rate – Collects KYZ pulse points from the signal transmitter

    • If the signal transmitter is not an RTU-LMI or LCU, the following scan category is displayed:

      • Periodic Health Check – Periodically checks that the signal transmitter is alive

When the check box of a scan category is selected, the following scan options are enabled:

    1. Select the default scan interval from the Interval drop-down list.

    2. Select the alternate scan interval from the Alt Interval drop-down list.

    3. Select a scan group from the Group drop-down list. The scan group is used to assign priorities to the scan categories.

  1. Under the Optional Scan Window, select the Enabled check box to define a time when the scans can occur.

    1. Select the type of scan from the Type drop-down list.

    2. Enter the time at which the scan window starts in the Open text box.

    3. Enter the time at which the scan window stops in the Close text box.

  2. Click the Next button.

  3. Select the comm channel for the signal transmitter from the Select a Communication Channel drop-down list.

The rest of this procedure depends on the type of comm channel selected.

    • If the selected comm channel that is Dedicated, click the Finish button.

The new signal transmitter appears in the Device Editor. For instructions on using this editor, refer to the related topics.

    • If the selected comm channel is Dialup:

      1. Click the Next button.

      2. Enter the phone number that is used to dial the signal transmitter in the Phone Number text box.

      3. Click the Finish button.

The new signal transmitter appears in the Device Editor. For instructions on using this editor, refer to the related topics.
