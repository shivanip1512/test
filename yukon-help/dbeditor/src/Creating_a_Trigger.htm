Creating a Trigger

To create a new trigger:

 

  1. Select either Threshold or Status from the Type drop-down list.

  2. Select the device to use for this trigger from the Device drop-down list.

  3. Select the point to use for this trigger from the Point drop-down list.

  4. If you selected Status as the trigger type, select a state from the Normal State drop-down list. This is the normal state of the point selected in step 3.

  5. If you selected Threshold as the trigger type:

    1. Enter a threshold and minimum restore offset in the Threshold and Min Restore Offset text boxes. The threshold minus the minimum restore offset is the level that the load must be below before a program can stop controlling.

    2. An optional threshold kickup offset can be entered in the ATKU text box. If the peak load value exceeds the threshold, the threshold is automatically increased to the new peak minus this offset. A value of zero disables the automatic threshold kickup offset.

    3. Enable or disable peak tracking by selecting or clearing the Use Peak Tracking check box. Peak tracking records the highest peak value.

    4. If peak tracking is enabled, select a device to use for peak tracking from the Device drop-down list and select a point to use for the peak tracking from the Point drop-down list.

  6. To setup trigger data projection, click the Projection button. This projects the value of the trigger after a set number of minutes. If the projected value is above the threshold, Yukon starts to control.

    1. Select the type of projection from the Type drop-down list. The types are (none) or LSF (Least Squared Fit).

    2. Select the number of samples to use in the LSF projection from the Samples drop-down list.

    3. Select how far ahead in time to project from the Ahead drop-down list.

    4. Click OK to save the control area projection.

  7. Click OK to save the trigger.

    • If only one threshold trigger exists and the threshold value is surpassed, control is performed on the next priority level.

    • If only one status trigger exists and the status changes, control is performed on the all of the priority levels.

    • If both a threshold and status trigger exist, and the Require All Triggers Active check box is selected, then control is performed only if the threshold is surpassed and the status changes. Control is performed on the next priority level.

    • If both a threshold and status trigger exist, and the Require All Triggers Active check box is not selected, then control is performed if either the threshold is surpassed or the status changes.
