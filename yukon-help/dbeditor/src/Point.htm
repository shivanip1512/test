Point

A point may be any of the following types:

  • Analog - examples include a Voltage or Amperage transducer on an RTU or a pseudo point.

  • Status - examples include switch status (open or closed) or status point residing on a foreign system. A status point can also have control.

 

A device name and point name are used together to access data that is attached to hardware in the field. When referring to one of these points, the system always uses both the device name and point name.

 

Therefore, meter readings for the kWh point will appear as:

 

JONES HOUSE KWH = xxx

 

In the case of a larger device, like an RTU, there may be many points of the same type. For example, an RTU named EAST SUB might have the following analog points:

  • FEEDER 1 PHASE A AMPS

  • FEEDER 1 PHASE B AMPS

  • FEEDER 1 PHASE C AMPS

 

In this case, the point number field tells the system which physical point on the device terminal block each input is wired to.

 

For status points, you should also create a State Name definition which assigns meaningful text messages to each of the status positions, for example: “Failed” and “Normal.”
