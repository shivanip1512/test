Creating a Load Program

One or more load programs can be defined in the database. A load program defines how the load is controlled, the load groups to control, and the customers to notify. When a load program is created, a status point is automatically added to the load program. The status point keeps track of when the load program is controlling and when it is stopped.

 

Perform the following steps to create a load program:

 

  1. From the View menu, select Load Management.

  2. From the Create menu, select Load Program.

  3. Enter a name for the load program in the Name text box.

  4. Select an operational state from the Operational State list.

    • Automatic – The program controls when the trigger threshold is exceeded or when the program is manually started.

    • Manual Only – The program must be manually started to perform control.

    • Timed – The program controls during the control windows specified in step 11.

  5. Select the constraint to use from the Program Constraint drop-down list. A program constraint controls when the load program can run.

  6. To specify a trigger offset for this program, enter an offset in the Trigger Offset text box. When the load program is assigned to a control area, the trigger offset is added to the trigger threshold. The trigger point defined in the control area must exceed the trigger offset plus the trigger threshold before the load program can start control. The default value is zero.

  7. To specify a restore offset for this program, enter an offset in the Restore Offset text box. When the load program is assigned to a control area, the restore offset is subtracted from the control area trigger threshold. The trigger point defined in the control area must go below the control area trigger threshold minus the trigger offset before the load program can stop control. A restore offset with a positive value causes the trigger threshold to be lowered; while a restore offset with a negative value causes the trigger threshold to be raised. The default value is zero.

  8. Click the Next button.

  9. Click the Add button to create a gear for the load program. Gears define the different rates at which a load program can controls loads. Different gear ratios result in different amounts of reduction. To continue creating a gear, click on the type of gear that to create from the list of gear types below:

    • Time Refresh  – The Time Refresh gear is used for irrigation control, C&I control, and warnings/notifications. The gear controls load control devices by sending shed commands, with a fixed or dynamic shed time, to one or more load groups. The Command Resend Rate setting specifies how often the shed command is resent and the How to Stop Control setting specifies how control is stopped.

    • Rotation – The Rotation gear is used to control legacy load control devices. The gear controls load control devices by sending shed commands, with a fixed shed time, to a rotating list of load groups. The Command Resend Rate setting specifies how often the shed command is resent and How to Stop Control setting specifies how control is stopped.

    • Master Cycle – The Master Cycle gear is used to cycle load control devices that cannot perform cycling themselves. The gear controls load control devices by sending cycling commands to one or more load groups. The Control Percent and Cycle Period settings specify the percentage of control over a period of time and the How to Stop Control setting specifies how control is stopped.

    • Smart Cycle – The Smart Cycle gear is used to control air conditioners (AC) and water heaters attached to load control devices that can perform the cycling themselves. The gear controls load control devices by sending cycling commands to one or more load groups. The Control Percent and Cycle Period settings specify the percentage of control over a period of time. The Command Resend Rate setting specifies how often the cycle command is resent and the How to Stop Control setting specifies how control is stopped.

    • TrueCycle – The TrueCycle™ gear is used to control air conditioners attached to load control devices that can adjust the cycling percentage based on the AC compressor runtime for the hour before control and the historical hourly runtime profile of the air conditioner. The gear controls load control devices by sending cycling commands to one or more load groups. The Control Percent and Cycle Period settings specify the percentage of control over a period of time. The Command Resend Rate setting specifies how often the cycle command is resent and the How to Stop Control setting specifies how control is stopped.

    • Magnitude Cycle – The Magnitude Cycle gear is used to control air conditioners with variable speed compressors or Direct Digital Controls (DDC) that are attached to load control devices that can measure the magnitude of the AC compressor's current. The magnitude measurement is combined with historical data collected by the LCR to generate a historical runtime for the AC unit. The gear controls load control devices by sending cycling commands to one or more load groups. The Control Percent and Cycle Period settings specify the percentage of control over a period of time. The Command Resend Rate setting specifies how often the cycle command is resent and the How to Stop Control setting specifies how control is stopped.

    • Target Cycle – The Target Cycle gear is used to control air conditioners attached to load control devices, which can reduce kW usage by the amount specified in the gear. The gear controls load control devices by sending cycling commands to one or more load groups. The Control Percent and Cycle Period settings specify the percentage of control over a period of time. The Command Resend Rate setting specifies how often the cycle command is resent and the How to Stop Control setting specifies how control is stopped.

    • Latching – The Latching gear is used to control a point on an RTU. The gear controls RTUs by sending status point control commands to one or more load groups. The Control Start State setting specifies the starting state of the control point.

    • Thermostat Ramping – The Thermostat Ramping gear is used to control air conditioner(s) by performing thermostat ramping. The gear controls the setpoint of a thermostat by sending setpoint control commands to one or more load groups. The Deltas/Absolute setting specifies whether the temperature setpoint values are delta values and absolute values. The Fahrenheit/Celsius setting specifies whether the temperature setpoint values are in degrees Fahrenheit or Celsius. The Ta through Tf values specify the times of the day that the temperature setpoint is changed and the How to Stop Control setting specifies how control of the thermostat is stopped.

    • Simple Thermostat Ramping – The Simple Thermostat Ramping gear is used to control air conditioner(s) by performing expanded thermostat ramping. The gear controls the setpoint of a thermostat by sending setpoint control commands to one or more load groups. The time periods of the gear are adjusted based on the control duration entered by the user. The Max Runtime specifies the maximum runtime of the load program, the Resend Rate setting specifies how often the setpoint control command is resent, and the How to Stop Control setting specifies how control is stopped.

    • No Control – The No Control gear is used to trigger notifications without performing control.

  1. Click the Next button.

  2. To specify a control window, select the Use Window 1 check box and select a start time for the control window from the Hr and Min drop-down lists in the Start area and select a stop time for the control window from the Hr and Min drop-down lists in the Stop area.

To specify a second control window, select the Use Window 2 check box and select a start time for the control window from the Hr and Min drop-down lists in the Start area and select a stop time for the control window from the Hr and Min drop-down lists in the Stop area.

  1. Click the Next button.

  2. To assign a load group to the load program, select the load group from the Available Groups list and click the Add button. To select more than one load group, hold the Ctrl key on the keyboard while clicking additional load groups. To rearrange the order of the assigned load groups, click and drag a load group to move it up or down in the list.

To unassign a load group to the load program, select the group from the Assigned Groups list and click the Remove button. To select more than one load group, hold the Ctrl key on the keyboard while clicking additional groups.

  1. Click the Next button.

  2. Perform the following steps to specify whether or not to use notification messages and to assign notification groups to the load program:

    • To send out a notification message before the load program starts, select the Program Start check box and enter how many minutes before the load program starts to send the message. If this check box is not selected, a notification message is not sent out when the load program starts.

    • To send out a notification message after the load program stops, select the Program Stop check box and enter how many minutes after the load program stops to send the message. If this check box is not selected, a notification message is not sent out when the load program stops.

    • To assign a notification group to the load program, select the notification group from the Notification Groups list and click the Add button. To select more than one notification group, hold the Ctrl key on the keyboard while clicking additional notification groups.

    • To unassign a notification group from the load program, select the notification group from the Assigned Groups list and click the Remove button. To select more than one notification group, hold the Ctrl key on the keyboard while clicking additional notification groups.

  1. Click the Finish button. The new load program is displayed in the Load Program Editor. For instructions on editing a load program, refer to the related topics.
