Editing a Trigger

Up to two triggers can be defined per control area.

 

  • If only one threshold trigger exists and the threshold value is surpassed, control is performed on the next priority level.

  • If only one status trigger exists and the status changes, control is performed on the all of the priority levels.

  • If both a threshold and status trigger exist, and the Require All Triggers Active check box is selected, then control is performed only if the threshold is surpassed and the status changes. Control is performed on the next priority level.

  • If both a threshold and status trigger exist, and the Require All Triggers Active check box is not selected, then control is performed if either the threshold is surpassed or the status changes.

The following changes can be made to a trigger:

 

  1. Select the trigger to edit from the Trigger drop-down list.

  2. A different type of trigger can be selected from the Type drop-down list.

    • Status - control occurs on a status change.

    • Threshold - control occurs when the threshold is surpassed.

  3. A different device on which the trigger point is located can be selected from the Device drop-down list.

  4. A different point to use for this trigger can be selected from the Point drop-down list.

  5. If you selected Status as the trigger type, a different state can be selected from the Normal State drop-down list. This is the normal state of the point selected in step 4.

  6. If you selected Threshold as the trigger type:

    1. The threshold and minimum restore offset can be changed in the Threshold and Min Restore Offset text boxes. The threshold minus the minimum restore offset is the level that the load must be below before a program can stop controlling.

    2. The threshold kickup offset can be changed in the ATKU text box. If the peak load value exceeds the threshold, the threshold is automatically increased to the new peak minus this offset. A value of zero disables the automatic threshold kickup offset.

    3. Enable or disable peak tracking by selecting or clearing the Use Peak Tracking check box. Peak tracking records the highest peak value.

    4. If peak tracking is enabled, select a device to use for peak tracking from the Device drop-down list and select a point to use for the peak tracking from the Point drop-down list.

  7. To setup trigger data projection, click the Projection button. This projects the value of the trigger after a set number of minutes. If the projected value is above the threshold, Yukon starts to control.

    1. Select the type of projection from the Type drop-down list. The types are (none) or LSF (Least Squared Fit).

    2. Select the number of trigger values to use in the LSF projection from the Samples drop-down list. If the trigger values in the projection are older than one hour, the current trigger value is used.

    3. Select how far ahead in time to project from the Ahead drop-down list.

    4. Click OK to save the control area projection.

  8. Click OK to save the changes.
