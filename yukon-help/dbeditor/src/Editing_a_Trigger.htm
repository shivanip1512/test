Editing a Trigger

Up to two triggers can be defined per control area.

  • If only one threshold trigger exists and the threshold value is surpassed, control is performed on the next priority level.

  • If only one status trigger exists and the status changes, control is performed on the all of the priority levels.

  • If both a threshold and status trigger exist, and the Require All Triggers Active check box is selected, then control is performed only if the threshold is surpassed and the status changes. Control is performed on the next priority level.

  • If both a threshold and status trigger exist, and the Require All Triggers Active check box is not selected, then control is performed if either the threshold is surpassed or the status changes.

The following changes can be made to a trigger:

 

  1. Select the trigger to edit from the Trigger drop-down list.

  2. A different type of trigger can be selected from the Type drop-down list.

    • Threshold – Control is initiated when the value of the trigger point exceeds the specified threshold.

    • Status – Control is initiated when the status of the trigger point changes from the specified normal state to a different state.

  3. A different device, which contains the point to use for the trigger, can be selected from the Device drop-down list in the Trigger Identification area.

  4. A different point to use for the trigger can be selected from the Point drop-down list in the Trigger Identification area.

  5. If Status was selected as the trigger type in step 2, a different normal state for the point selected in step 4 can be selected from the Normal State drop-down list.

  6. If Threshold was selected as the trigger type in step 2:

    1. The kW threshold and minimum restore offset can be changed in the Threshold and Min Restore Offset text boxes. The threshold minus the minimum restore offset is the kW level that the load must be below before a program can stop controlling.

    2. The kW threshold kickup offset can be changed in the ATKU (Automatic Threshold Kickup Offset) text box. If the peak load value exceeds the threshold, the threshold is automatically increased to the new peak load value minus this offset. A value of zero disables the Automatic Threshold Kickup Offset.

    3. Enable or disable peak tracking by selecting or clearing the Use Peak Tracking check box. Peak tracking records the highest peak load value.
      If peak tracking is enabled, select a device to use for peak tracking from the       Device drop-down list and select a point to use for the peak tracking from the Point drop-down list.

  7. To setup trigger data projection for a threshold trigger, click the Projection button. The Yukon system projects what the value of the trigger will be after a specified number of minutes. If the projected value is above the threshold, the Yukon system initiates control.

    1. Select the type of data projection from the Type drop-down list. The types are (none) or LSF (Least Squared Fit).

    2. Select the number of trigger values to use in the LSF projection from the Samples drop-down list.
      Note:       If the trigger values in the projection are older than one hour, the current trigger value is used.

    3. Select the number of minutes ahead in time to project values from the Ahead drop-down list.

    4. Click the OK button to save the trigger data projection.

  8. Click the Apply button to save the changes to the trigger or click the OK button to save the changes to the trigger and close the Control Area Editor.
