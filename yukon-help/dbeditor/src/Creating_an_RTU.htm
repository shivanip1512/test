Creating an RTU

Perform the following steps to create an RTU (Remote Terminal Unit):

 

  1. From the View menu, select Core.

  2. From the Create menu, select Device.

  3. Select RTU from the list on the left.

  4. Select the type of RTU to create from the list on the right:

    • RTU-DNP – Remote Terminal Unit that supports the DNP3 protocol.

    • RTU-MODBUS – Remote Terminal Unit that supports the Modbus/RTU protocol.

    • RTU-WELCO – Remote Terminal Unit that supports the ESCA/WELCO protocol.

    • RTM – Remote Telemetry Module that supports either the DNP3 or Modbus protocol.

  5. Click the Next button.

  6. Enter a name for the RTU in the Device Name text box.

  7. Enter the address of the RTU in the address text box.

    • If RTU-DNP or RTU-MODBUS was selected in step 4, enter a slave address in the Slave Address text box.

  • If RTU WELCO or RTM was selected in step 4, enter a physical address in the Physical Address text box. The physical address is programmed into the device.

  1. Click the Next button.

  2. Select the check box of the scan categories to enable. The scan categories vary depending on the type of RTU selected in step 4.

  • If the RTU is an RTU-DNP, the following scan categories are displayed:

      • Class 0, 1, 2, 3 Scan – Collects all of the data from the RTU.

      • Class 1, 2, 3 Scan – Collects the data from the RTU that has changed since the last scan.

  • If the RTU is an RTU-MODBUS, the following scan category is displayed:

      • Integrity Rate – Collects all point values from the RTU.

  • If the RTU is an RTU-WELCO, the following scan categories are displayed:

      • Integrity Rate – Collects all point values from the RTU.

      • General Scan – Collects general data from the RTU.

      • Accumulator Rate – Collects KYZ pulse points from the RTU.

  • If the RTU is an RTM, the following scan category is displayed:

      • General Scan – Collects general data from the RTU.

When the check box of a scan category is selected, the following scan options are enabled:

    • Interval – Default scan interval of the selected scan category.

    • Alt Interval – Alternate scan interval of the selected scan category.

    • Group – Scan group of the selected scan category. The scan group is used to assign priorities to the scan category.

  1. Under the Optional Scan Window, select the Enabled check box to define a time when the scans can occur.

    1. Select the type of scan from the Type drop-down list.

    2. Enter the time at which the scan window starts in the Open text box.

    3. Enter the time at which the scan window stops in the Close text box.

  2. Click the Next button.

  3. Select the comm channel for the RTU from the Select a Communication Channel drop-down list.

The rest of this procedure depends on the type of comm channel selected.

    • If the selected comm channel is Dedicated, click the Finish button.

The new RTU appears in the Device Editor. For instructions on editing an RTU, refer to the related topics.

    • If the selected comm channel is Dialup or Dialback:

    1. Click the Next button.

    2. Enter the phone number used to dial this RTU or dialback from this RTU in the Phone Number text box.

    3. Click the Finish button.

The new RTU appears in the Device Editor. For instructions on editing an RTU, refer to the related topics.
