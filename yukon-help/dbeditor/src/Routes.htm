Routes

A route is defined as a communication path through the distribution system to a field device. These paths are used by the system to control the number of communications distributed on the system.

 

Before a communication route can be defined, one or more signal transmitter devices must be defined. Examples include:

  • CCU

  • LCU

  • Paging Tap Terminal

  • WCTP Terminal

  • TCU

 

For a Power Line Carrier, where the communication route is unable to reach its entire region, a signal repeater route must be defined. Routes defined using repeaters originate from the transmitter and have their signal rebroadcast by the repeater.

 

Finally, macro routes are used to combine multiple individual routes into a combination route. Macros are used for sending load management as one-way messages to multiple areas of a system. They can also be used with MCTs as a list of possible communication paths.

 

NOTE Whenever a signal transmitter is added to the Yukon Editor, the Editor automatically adds a route for this transmitter to the database.
