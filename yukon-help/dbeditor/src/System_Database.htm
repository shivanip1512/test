System Database

The System database tracks the customer, contact, notification group, and holiday information for a Yukon system.

 

Cannon Technologies recommends creating a new system database in the following order:

  1. Define the contacts. Contacts are people who can be notified of the need for curtailment or other events.

  2. Define the customers. Customers are commercial or industrial users who have signed up for load management programs.

  3. Define the logins. Logins control user privileges for Yukon access.

  4. Define the login groups. Login groups control group privileges for Yukon access.

  5. Define the notification groups. A notification group controls who receives notifications and how the information is sent.

  6. Define the holiday schedules. Holiday Schedules contain the dates of holidays for the year.

  7. Define the season schedules. Season schedules allow you to define when load control can occur. Season schedules contain the start and end dates of load control seasons.

  8. Define the Time-Of-Use (TOU) schedules. TOU schedules define the rates to use and when to use the rates.

  9. Define the baselines. A baseline is a known value that is used for comparison with current values.

  10. Define the tags. Tags can be used to inhibit control of field devices.
