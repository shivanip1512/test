Point Limits Tab

The following changes can be made to the point limits settings:

 

  1. If the point is not a status point, the high and low limits can be enabled or disabled by selecting or clearing the Limit 1 and/or Limit 2 check boxes.

    • If Limit 1 is enabled, enter a high and low value for the limit in the High and Low text boxes. If the value goes outside of this range, the limit is violated and an alarm is generated.
      The limit duration of Limit 1 can be changed in the Limit Duration 1 spin box. This specifies the number of seconds that Limit 1 must be exceeded before an alarm is generated.

    • If Limit 2 is enabled, enter a high and low value for the limit in the High and Low text boxes. If the value goes outside of this range, the limit is violated and an alarm is generated.
      The limit duration of Limit 2 can be changed in the       Limit Duration 2 spin box. This specifies the number of seconds that Limit 2 must be exceeded before an alarm is generated.

  2. To enable checking for stale data, select the Enable check box in the Stale Data area.

    1. Select the length of time, in minutes, that a data point must be non-updated or bad to be considered stale from the Time in Minutes drop-down list.

    2. Select the type of point update on which check if the data point is stale from the Update Style drop-down list. The type of point updates are:

      • Always – Checks if the data point is stale on any point change.

      • On Change – Checks if the data point is stale when an operand causes the point to change.

  3. If the point is not a status point, the reasonability limits can be enabled or disabled by selecting or clearing the High Limit and/or Low Limit check boxes.

    • If high limit is enabled, enter the high reasonability limit value in the High Limit text box.

    • If low limit is enabled, enter the low reasonability limit value in the Low Limit text box.

If a reading exceeds a high or low reasonability limit, the reading is ignored and the last reasonable value is used instead.
